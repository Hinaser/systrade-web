API Reference

In SysTrade, you can compose bitcoin trade algorithm by Javascript.
Here I introduce how to write actual programmed order and available APIs.

All APIs described here can be executed in `Machine Order` tab.
For example, if you write `trader.buy(2)` and execute in Machine Order tab, an order to buy 2 BTC for market price will be requested.

Shared variables

Every time you execute a program in Machine Order tab, new execution environment is created.
This means local variables declared in previous execution cannot be accessed in next execution.
But don't worry. You can bring those local variables into next execution context by assigning the values into `exports` object as following.

exports["name"] = value;

exports Object is shared among whole execution contexts.

API
trader

You can request an order, settle positions, cancel orders with `trader` instance.

trader.buy(amount, [price])

Request a bid order.

example
trader.buy(2, 1000000);
Return value
type description
object Object having order id. The id is used to cancel order or find order.
Arguments
arg Is required？ type description
Amount Required number Specify bit coin amount you want to buy. The unit is BTC. Minimum amount is 0.001 BTC. Keep in mind that if amount is more than or equal to 0.001 BTC, you can specify as small digit as 1 satoshi (10e-8 BTC).
i.e. 0.00100001 BTC
Price Optional number Specify order price of bitcoin. If you omit this argument, order is requested as market order.

Request an ask order.

example
trader.sell(2, 1000000);
Return value
type description
object Object having order id. The id is used to cancel order or find order.

arg Is required？ type description
amount Required number Specify bit coin amount you want to buy. The unit is BTC. Minimum amount is 0.001 BTC. Keep in mind that if amount is more than or equal to 0.001 BTC, you can specify as small digit as 1 satoshi (10e-8 BTC).
i.e. 0.00100001 BTC
price Optional number Object having order id. The id is used to cancel order or find order.

example

Return value
type description
Promise Promise will be resolved when response of cancel request is returned by market server.
Arguments
arg Is required？ type description

Return value
type description

trader.getMarketState()

Get market server status.

example
trader.getMarketState();
Return value
type description
object Get server load and board state.
{
  health: "NORMAL",
  state: "RUNNING"
}
The values of health/state are as follows: (source: https://lightning.bitflyer.jp/docs/api) 
health: The load status of market server. Value is one of the followings:
  NORMAL: Server is running normally.
  BUSY: Server is running but overloaded.
  VERY BUSY: Server load is heavy.
  SUPER BUSY: Server load is too heavy. Orders might be failed or delayed.
  NO ORDER: Server cannot accept orders.
  STOP: Server is not running. Any order will not be accepted.

state: The board state.
  RUNNING: Board is running normally
  CLOSED: Trade is now closed.
  STARTING: Restarting board.
  PREOPEN: In itayose
  CIRCUIT BREAK: Circuit breaker is running
  AWAITING SQ: After Lightning Futures trade market is closed and before SQ is committed.
  MATURED: Lightning Futures has reached mature.
  data: Additional data
  special_quotation: SQ of Lightning Futures
Arguments
No arguments

trader.getExecutions()

Get history of executions.

example
trader.getExecutions();
Return value
type description
Array Get an array of executions history.
The format of object array is as follows:
[
  {
    "id": 37233,
    "child_order_id": "JOR20150707-060559-021935",
    "side": "BUY",
    "price": 33470,
    "size": 0.01,
    "commission": 0,
    "exec_date": "2015-07-07T09:57:40.397",
    "child_order_acceptance_id": "JRF20150707-060559-396699"
    "ext": {status: "UNCONFIRMED"} // Please see below
  },
  ...
]
The value of ext.status is either "UNCONFIRMED" or "CONFIRME".
UNCONFIRMED means diff of position size which should be made by the execution is not confirmed yet.
For example, if there is a bid execution of 2.1 BTC, there must be position size change associated with the execution.
That is, after execution bid position size increases by 2.1 BTC, or ask position size decreases by 2.1 BTC. When SysTrade confirms those position size move, status turns to be "CONFIRMED".
Arguments
No arguments
monitor

An instance of market monitor which stores chart, board, execution data.

monitor.getChart(minuteFoot, index, type)

Get value from candlestick chart

example
trader.getChart("1", "latest", "max");
Return value
type description
number Get value from designated chart type. i.e. The max price in the latest 1 min chart.
Arguments
arg Is required？ type description
minuteFoot Required string Specify chart minute value. By default one of "0.2", "1", "3", "5", "15" is available.
index Required number | string Specify index of candle stick from the latest candlestick. 0 means the latest stick. 3 means 3 previous stick. "latest" is equal to 0.
type Required string Specify type of candlestick data.
Possible value is one of "max", "min", "start", "last", "volume", "size", "avg", "diff".
Each represents "highest price", "lowest price", "opening price", "closing price", "total trade volume in JPY", "total trade volume in BTC", "average price", "Difference between closing and opening price" of a designated candle chart.
monitor.SMA(minuteFoot, type, numOfSticks)

Get simple moving average of candle stick chart.

example
monitor.SMA(3, "volume", 10);
Return value
type description
number Simple moving average value of specified chart data. i.e. SMA("1", "start", 10) gets sma of opening price of 1 minute chart among 10 sticks.
Arguments
No arguments

Sample code
Script to send a bid/ask order every 10 seconds.
var flag = true;
var interval = 10*1000; // 10000 milli seconds = 10 seconds

setInterval(function(){
  if(flag === true){
    trader.buy(2);
  }
  else{
    trader.sell(2);
  }
  flag = !flag;
}, interval);
