APIリファレンス

SysTradeではプログラミング言語(Javascript)を用いたビットコインのアルゴリズム取引が可能です。
ここでは具体的なプログラム注文の書き方、プログラムで利用できるAPIについて記載します。

ここに記載されているAPIは「プログラム注文」タブで全て実行可能です。
例えばプログラム注文タブで「trader.buy(2);」と入力し実行すると2BTCの 成行買い注文がリクエストされます。

データ共有

プログラム注文でプログラムを実行すると実行環境が毎回新しく作成されます。
これはプログラムを実行する度に以前に宣言した変数の値は次のプログラム実行時に 引き継がれないということです。
変数にセットした値を次回実行時のプログラムでも利用したい場合は

exports["任意のプロパティ名"] = 格納したい値;

のように exports オブジェクトを利用してください。
exportsオブジェクトは実行環境間で共有されます。

API
trader

traderインスタンスを使うと売買注文の発行、建玉決済、キャンセル等ができます。

trader.buy(数量, [価格])

買い注文を発行します。

例
trader.buy(2, 1000000);
戻り値
型説明
object注文idが含まれるオブジェクト。注文をキャンセルする時や、注文結果の確認等に使用します。
引数
引数必須？型説明
数量必須numberビットコインの数量を指定します。単位はBTCです。 最低数量は0.001BTCです。ただし、0.001BTC以上であれば 1satoshi単位まで細かい数量の指定が可能です。
(例) 0.00100001 BTC
価格任意numberビットコインの注文価格を指定します。省略した場合は成行注文になります。
trader.sell(数量, [価格])

売り注文を発行します。

例
trader.sell(2, 1000000);
戻り値
型説明
object注文idが含まれるオブジェクト。注文をキャンセルする時や、注文結果の確認等に使用します。
引数
引数必須？型説明
数量必須numberビットコインの数量を指定します。単位はBTCです。 最低数量は0.001BTCです。ただし、0.001BTC以上であれば 1satoshi単位まで細かい数量の指定が可能です。
(例) 0.00100001 BTC
価格任意numberビットコインの注文価格を指定します。省略した場合は成行注文になります。

trader.cancel(注文ID)

指定したIDの注文をキャンセルします。

例

戻り値
型説明
Promiseキャンセルリクエスト完了時にonFulfilled/onRejectedコールバックが実行されます。

引数必須？型説明

trader.getMarketState()

マーケットサーバの状態を取得します。

例
trader.getMarketState();
戻り値
型説明
objectサーバの負荷状況および板の状態を取得します。
{
  health: "NORMAL",
  state: "RUNNING"
}
health/stateの種類は下記の通りです。(引用元: https://lightning.bitflyer.jp/docs/api)
health: 取引所の稼動状態です。以下のいずれかの値をとります。
  NORMAL: 取引所は稼動しています。
  BUSY: 取引所に負荷がかかっている状態です。
  VERY BUSY: 取引所の負荷が大きい状態です。
  SUPER BUSY: 負荷が非常に大きい状態です。発注は失敗するか、遅れて処理される可能性があります。
  NO ORDER: 発注が受付できない状態です。
  STOP: 取引所は停止しています。発注は受付されません。

state: 板の状態です。以下の値をとります。
  RUNNING: 通常稼働中
  CLOSED: 取引停止中
  STARTING: 再起動中
  PREOPEN: 板寄せ中
  CIRCUIT BREAK: サーキットブレイク発動中
  AWAITING SQ: Lightning Futures の取引終了後 SQ（清算値）の確定前
  MATURED: Lightning Futures の満期に到達
  data: 板の状態について、付加情報を提供します。
  special_quotation: Lightning Futures の SQ（清算値）
※今後、表示項目について変更・追加の可能性はございます。予め了承ください。
引数
引数なし

trader.getExecutions()

約定履歴を取得します。

例
trader.getExecutions();
戻り値
型説明
Array約定履歴の配列を取得します。
約定履歴オブジェクトのフォーマットは下記をご参考ください。
[
  {
    "id": 37233,
    "child_order_id": "JOR20150707-060559-021935",
    "side": "BUY",
    "price": 33470,
    "size": 0.01,
    "commission": 0,
    "exec_date": "2015-07-07T09:57:40.397",
    "child_order_acceptance_id": "JRF20150707-060559-396699"
    "ext": {status: "UNCONFIRMED"} // 下記参照
  },
  ...
]
statusの値は"UNCONFIRMED"/"CONFIRMED"のいずれかになります。
UNCONFIRMEDは約定に対応する建玉の増減が未確認であることを示します。 例えば、2.1BTCの買いの約定があった場合、約定の前後で2.1BTC分 買い玉が増えるか、売り玉が減ることになります。この約定数量に対応する建玉の増減が 確認できた場合、statusが"CONFIRMED"になります。

monitor

マーケットのチャートデータ等を保持しているインスタンスです。

monitor.getChart(分足, 時点, タイプ)

価格のチャートを取得します。

例
trader.getChart("1", "latest", "max");
戻り値
型説明
number指定したチャートタイプの価格値を取得します。 (例) 1分足の最新の足の最高価格
引数
引数必須？型説明
分足必須stringチャートの分足を指定します。デフォルトでは0.2, 1, 3, 5, 15分足が指定できます。
時点必須number | string現在と比較して何本前の足のローソクデータを取得するか指定します。 "latest"は0と等価です。
タイプ必須string取得するローソク足データの種類を指定します。
"max", "min", "start", "last", "volume", "size", "avg", "diff"が指定できます。
それぞれ、高値、安値、始値、終値、出来高(円ベース)、総取引数量(BTCベース)、平均値、終値と始値の差を表します。
monitor.SMA(分足, 種類, チャート本数)

価格の単純移動平均を取得します。

例
monitor.SMA(3, "volume", 10);
戻り値
型説明
number指定したチャートデータの単純移動平均
例えばSMA("1", "start", 10)は、1分足チャートの過去10足(10分)の 始値の移動平均を取得します。
引数
引数なし

サンプルコード
10秒毎に買い注文、売り注文を交互に出すコード
var 売買フラグ = true;
var インターバル = 10*1000; // 10000ミリ秒 = 10秒

setInterval(function(){
  if(売買フラグ === true){
    trader.buy(2);
  }
  else{
    trader.sell(2);
  }
  売買フラグ = !売買フラグ;
}, 10*1000);
